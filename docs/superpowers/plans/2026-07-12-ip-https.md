# IP-Based HTTPS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Serve the Lightsail test deployment through browser-trusted HTTPS at `https://16.54.60.43` without a registered domain.

**Architecture:** Host Nginx terminates TLS and proxies to the existing Vite container on `127.0.0.1:5173`. Certbot obtains a short-lived Let's Encrypt IP certificate using HTTP-01 webroot validation and renews it automatically; Lightsail exposes only SSH, HTTP, and HTTPS after verification.

**Tech Stack:** Ubuntu 24.04 LTS, Nginx, Certbot 5.4+, Let's Encrypt ACME, systemd, Docker Compose, AWS Lightsail

## Global Constraints

- Public IP is `16.54.60.43`; no registered domain is used.
- Certificate profile is `shortlived`, with approximately 160-hour validity.
- Do not print, copy, or commit certificate private keys or `.env` secret values.
- Preserve the Docker volume `church-operation-app_mongo-data`.
- Keep public TCP `8080` and TCP `27017` blocked.
- Remove public TCP `5173` only after HTTPS verification passes.
- This remains a test deployment; real church data is prohibited.

### Task 1: Install And Stage Nginx And Certbot

**Files:**
- Create: remote `/var/www/letsencrypt/.well-known/acme-challenge`
- Create: remote `/etc/nginx/sites-available/church-operation`
- Modify: remote `/etc/nginx/sites-enabled/church-operation`

**Interfaces:**
- Consumes: running Vite service on `127.0.0.1:5173`
- Produces: HTTP reverse proxy and ACME webroot on port `80`

- [ ] **Step 1: Install Nginx and current Certbot snap**

Run remotely:

```bash
sudo apt-get update
sudo apt-get install -y nginx snapd
sudo snap install core
sudo snap refresh core
sudo snap install --classic certbot
sudo ln -sf /snap/bin/certbot /usr/local/bin/certbot
certbot --version
```

Expected: Certbot version is `5.4.0` or newer.

- [ ] **Step 2: Create the ACME webroot and initial Nginx configuration**

Write `/etc/nginx/sites-available/church-operation` with:

```nginx
server {
    listen 80 default_server;
    listen [::]:80 default_server;
    server_name 16.54.60.43;

    location ^~ /.well-known/acme-challenge/ {
        root /var/www/letsencrypt;
        default_type text/plain;
    }

    location / {
        proxy_pass http://127.0.0.1:5173;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

Create the webroot, enable the site, and disable the packaged default:

```bash
sudo install -d -m 0755 /var/www/letsencrypt/.well-known/acme-challenge
sudo ln -sf /etc/nginx/sites-available/church-operation /etc/nginx/sites-enabled/church-operation
sudo unlink /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl enable --now nginx
```

Expected: Nginx configuration test is successful and HTTP requests proxy to Vite.

- [ ] **Step 3: Verify the ACME path**

Run remotely:

```bash
printf 'acme-ready\n' | sudo tee /var/www/letsencrypt/.well-known/acme-challenge/readiness >/dev/null
curl --fail --silent http://16.54.60.43/.well-known/acme-challenge/readiness
sudo unlink /var/www/letsencrypt/.well-known/acme-challenge/readiness
```

Expected: output is `acme-ready`.

### Task 2: Issue And Install The IP Certificate

**Files:**
- Create: remote `/etc/letsencrypt/live/16.54.60.43/fullchain.pem`
- Create: remote `/etc/letsencrypt/live/16.54.60.43/privkey.pem`
- Modify: remote `/etc/nginx/sites-available/church-operation`

**Interfaces:**
- Consumes: working public HTTP-01 webroot from Task 1
- Produces: trusted TLS endpoint on host port `443`

- [ ] **Step 1: Request the production certificate**

Run remotely:

```bash
sudo certbot certonly \
  --non-interactive \
  --agree-tos \
  --email lovecapstone@gmail.com \
  --preferred-profile shortlived \
  --webroot \
  --webroot-path /var/www/letsencrypt \
  --ip-address 16.54.60.43
```

Expected: certificate files are created under `/etc/letsencrypt/live/16.54.60.43/`.

- [ ] **Step 2: Configure Nginx TLS and HTTP redirect**

Replace `/etc/nginx/sites-available/church-operation` with:

```nginx
server {
    listen 80 default_server;
    listen [::]:80 default_server;
    server_name 16.54.60.43;

    location ^~ /.well-known/acme-challenge/ {
        root /var/www/letsencrypt;
        default_type text/plain;
    }

    location / {
        return 301 https://16.54.60.43$request_uri;
    }
}

server {
    listen 443 ssl default_server;
    listen [::]:443 ssl default_server;
    server_name 16.54.60.43;

    ssl_certificate /etc/letsencrypt/live/16.54.60.43/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/16.54.60.43/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_session_timeout 1d;
    ssl_session_cache shared:TLS:10m;

    location / {
        proxy_pass http://127.0.0.1:5173;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

Validate and reload:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

Expected: Nginx reloads with no configuration error.

- [ ] **Step 3: Open the HTTPS firewall rules**

In the Lightsail **Networking** tab, add TCP `443` for any IPv4 and any IPv6 address.

Expected: both firewall tables list HTTPS or custom TCP `443`; the existing TCP `5173` rules remain until HTTPS verification passes.

- [ ] **Step 4: Verify certificate identity and HTTPS response**

Run locally:

```bash
curl --fail --silent --show-error --head https://16.54.60.43/
openssl s_client -connect 16.54.60.43:443 -verify_return_error </dev/null 2>/dev/null | openssl x509 -noout -subject -issuer -dates -ext subjectAltName
```

Expected: HTTPS returns `200`, certificate verification succeeds, and the subject alternative name contains `IP Address:16.54.60.43`.
